The Required Data Inputs

Group A: Inputs for Training the Model
You will use these files to teach the algorithm what makes a good nanoparticle.
- screening_data.tsv: This is your target label dataset. It contains the exact 1,440 combinations that the lab physically tested. You need the drug name, the excipient name, and the binary code (1 = success, 0 = failure).
- selected_drugs_smiles.tsv: The chemical text structures (SMILES) for the 16 drugs used in the physical tests.
- selected_excipients_smiles.tsv: The chemical text structures for the 90 excipients used in the physical tests.

Group B: Inputs for High-Throughput Inference
Once your model is trained and highly accurate, you will feed it these massive files to discover new nanoparticles.
- drugbank_selfaggs_smiles.tsv: The SMILES strings for the 788 candidate drugs predicted to naturally self-aggregate.
- gras_iig.tsv: The SMILES strings for the 2,686 FDA-approved excipients.

The Project Flow
Here is the step-by-step logic you need to code to replicate the Nature Nanotechnology paper.

Step 1: Data Ingestion (ETL)
The Code: Write a Python script using the pandas library to load screening_data.tsv.
The Goal: Create a clean master table (DataFrame) where each row represents one physical experiment. Your columns should look something like: Drug_Name, Excipient_Name, and Target_Label (1 or 0).

Step 2: Feature Extraction (The Chemistry to Math Translation)
The Code: Use the RDKit library in Python.
The Goal: Machine learning models cannot read "SMILES" text; they need numbers. You will write a function that takes a SMILES string (like "CCO" for ethanol) and calculates its physicochemical properties (e.g., molecular weight, number of hydrogen bond donors, topological polar surface area). Do this for every drug in selected_drugs_smiles.tsv and every excipient in selected_excipients_smiles.tsv.

Step 3: Feature Concatenation (Building the Input Matrix)
The Code: pandas merge operations.
The Goal: For every row in your master table from Step 1, you need to combine the features of the drug and the features of the excipient. If a drug has 10 features and an excipient has 10 features, your final input array for that specific pairing will be 20 features long. This combined vector, paired with the Target_Label (1 or 0), is what you feed the algorithm.

Step 4: Model Training and Validation
The Code: Use scikit-learn to initialize a RandomForestClassifier.
The Goal: Split your 1,440 rows into a training set (e.g., 80%) and a testing set (e.g., 20%). Train the Random Forest on the 80%, and then ask it to predict the 20%. Calculate its accuracy, precision, and recall. If the numbers are strong, your model successfully understands the underlying chemistry!

Step 5: The Virtual Screen (Inference)
The Code: Loops and batch processing.
The Goal: Now, take the massive lists from Group B (drugbank_selfaggs_smiles.tsv and gras_iig.tsv). Create every possible pair between the 788 drugs and 2,686 excipients (giving you over 2.1 million combinations). Run your RDKit feature extraction on them, concatenate the features, and ask your trained Random Forest model to predict which ones will form nanoparticles.
